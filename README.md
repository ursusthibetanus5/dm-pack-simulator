# デュエマ パック開封シミュレータ

1カートン(12箱)を生成し、その中から無作為に5パックを開封する静的Webアプリです。

## 仕様

- 開封する弾をチェックボックスで選択し、弾ごとに開封パック数を指定(複数弾の組み合わせも可能、**合計5パックまで**)
- 選んだ弾ごとに1カートンを生成し、その中から指定パック数を無作為に開封
- 1カートン = 12箱、1箱 = 30パック、1パック = 5枚
- パックの基本封入は **C, C, R, UC, UC**
- 各箱の確定封入: **OR 1 / SR 4 / VR 6 / キャラプレミアムトレジャー 1 / 銀トレジャー 1 / 黒トレジャー 12 / R 16 / シク 1**
  - **オーバーレア 1枚** / **SR 4枚(箱内で種類被りなし)** / **キャラプレミアムトレジャー 1枚** / **銀トレジャー 1枚** / **VR 6枚(箱内で種類被りなし)** / **シク枠 1枚** — それぞれ別のパックのR枠と入れ替わる(R枠を潰す)
  - **R は毎弾16種類で、1箱で被りなく全種類が揃う**(残り16のR枠に16種類が1枚ずつ入る。そのため各弾の `pool.R` は16種類にすること)
  - **黒トレジャー 12枚** — UC1枚と入れ替わる。R枠とは独立に封入されるため、SR等のR以上と同じパックに共存し得る
  - **Cが3枚封入されるパックが箱に2つある**(そのパックはUCが1枚少ない)。
    3C封入パックは**R枠が素のRのパックにのみ存在**し、黒トレジャーも入らない。
    `config: { tripleCPacksPerBox: 2 }` で変更可能
  - **コモンは25種62枚・全種が必ず2枚か3枚出る(4枚は揃わない)** — C枠は1箱62枚
    (2枚×30パック+3C封入2パック)で、全種2枚+ランダム12種に3枚目が入る。
    各弾の `pool.C` は25種にすること
  - **アンコモンは25種・全種が最低1枚・最大3枚出る(0枚の種類は存在しない)** —
    UC枠は3C封入パックと黒トレジャーで潰れる分を除いた46枚で、全種1枚+残り21枚を
    ランダムな種類(上限3枚)で埋める。各弾の `pool.UC` は25種にすること。
    金トレジャー/ドリームレアは箱に2枚以上ある種類のUCとだけ入れ替わるため、
    どの箱でもUC全種が必ず1枚以上出る
  - **UCが1枚もないパックは存在しない**(例外なし) — 3C封入パックに黒は入らず、
    金/ドリームは黒入り・3C封入パックを避けるため、全パックにUCが最低1枚入る
  - **シク枠** — どのレアリティのカードがシクになるかは弾ごとの重み付き確率(`secret.weights`)で
    抽選され、対象カードは `secret.cards` にレアリティごとに全種登録する
- カートン単位の封入:
  - **金トレジャー** — 弾ごとに設定した枚数(`goldPerCarton`)がカートン全体のどこかに封入される。
    カートン内で種類は被らない。もう1つのUC枠と入れ替わるため、R以上や黒トレジャーとも共存し得る
  - **ドリームレア** — 弾ごとに設定した枚数(`dreamPerCarton`)がカートン単位で封入される。
    UC枠と入れ替わるが、**金トレジャーと同じ箱には封入されない**。カートン内で種類は被らない
- 表示はカード名とレアリティのみ(画像・能力テキストなし)

## カードデータの差し替え・弾の追加

カード名・封入数の設定はすべて `cards.js` にあります。収録済みの実在弾:

- **DM26-RP2** — DM1種/OR1種/SR11種/VR11種/R16種/UC25種/C25種/シク24種/
  キャラプレミアム5種/金5種/銀9種/黒10種。金トレ0.875枚・ドリームレア4.125枚/カートン、
  シク比率は8カートン実測値
- **DM26-RP1 逆札篇第1弾 逆転神VS切札竜** — 構成はDM26-RP2と同じ
  (シークレットドリームレア・シリアル版はシク枠の対象外として未収録)。
  金トレ/ドリームレアの封入数とシク比率はDM26-RP2の値を暫定流用
- **DM25-RP1 王道W第1弾 邪神vs邪神** — 構成はDM26-RP2と同じ
  (シークレットドリームレアはシク枠の対象外として未収録)。
  金トレ/ドリームレアの封入数とシク比率は実測データが無いためDM26-RP2の値を暫定流用
  (`config` と `secret.weights` で調整可能)

- **弾を追加する**: `SETS` 配列に `{ id, name, config, pool }` の要素を足す
- **実在の弾に差し替える**: 各弾の `pool` のレアリティ配列を実際のカード名にする
- **封入数を変える**: 全弾共通は `DEFAULT_CONFIG`、特定の弾だけ変えたい場合はその弾に
  `config: { srPerBox: 5 }` のように上書き設定を書く
- **金トレジャー/ドリームレアの封入数**: 弾ごとに `config: { goldPerCarton: 1, dreamPerCarton: 0.5 }`
  のように1カートンあたりの枚数を指定。端数は確率になる(`2.5` = 2枚確定+50%で3枚目、
  `0.5` = 2カートンに1枚相当。0なら封入なし)。カードは `pool.GOLD` / `pool.DREAM` に登録する
- **シク枠**: 弾ごとに以下のように設定する

  ```js
  secret: {
    weights: { SR: 2, VR: 3, R: 5 },  // シクになるレアリティの比率(この例: SR20%/VR30%/R50%)
    cards: {                          // シク仕様が存在するカードを全種登録
      SR: ["カード名", ...],
      VR: ["カード名", ...],
      R:  ["カード名", ...],
    },
  },
  ```

  `weights` に無いレアリティ・カード未登録のレアリティからはシクが出ない。
  箱あたりの枚数を変えたい場合は `config: { secretPerBox: 2 }` のように指定する

## GitHub Pages での公開手順

1. GitHubで新しいリポジトリを作成(例: `dm-pack-simulator`)
2. このフォルダの中身をpush:

   ```
   git init
   git add .
   git commit -m "デュエマ パック開封シミュレータ"
   git branch -M main
   git remote add origin https://github.com/<ユーザー名>/dm-pack-simulator.git
   git push -u origin main
   ```

3. リポジトリの **Settings → Pages** を開き、
   **Source: Deploy from a branch / Branch: main / フォルダ: / (root)** を選んで保存
4. 数分後に `https://<ユーザー名>.github.io/dm-pack-simulator/` で公開されます

ビルド不要の純粋な静的サイトなので、`index.html` をブラウザで直接開いてもローカルで動作確認できます。
